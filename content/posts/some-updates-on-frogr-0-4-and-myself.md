Too many things have happened to me during the last weeks (some good and some bad, as usual), but I still have found some time to continue improving <a title="Frogr web site" href="http://live.gnome.org/Frogr">frogr</a> towards the 0.4 release, which I hope it's gonna be, at least thinking of my very particular use cases, a very complete and functional release. Still, the UI won't be great (I'd really need help with this), I know that... but I said "complete and functional", right? I do not remember having mentioned "beautiful", "eye candy", or the like... that's a matter, though, I'll probably consider for following releases, but not for now.

The point of this post is that I declared yesterday the official start of the <a title="Mail in gnome-i18n mailing list" href="http://mail.gnome.org/archives/gnome-i18n/2011-January/msg00094.html">feature &amp; string freeze phase for frogr</a>, which basically means that what you can find in the <a title="NEWS file for frogr from master branch" href="http://git.gnome.org/browse/frogr/tree/NEWS">NEWS file</a> is exactly what you'll find in the next release (which should happen in 2-3 weeks time), and that the awesome people in the <a title="GNOME Translation teams" href="http://l10n.gnome.org/teams/">GNOME translations teams</a> can now work on adding new translations, or just updating the ones already present in the <a title="Damnes Lies of GNOME: frogr" href="http://l10n.gnome.org/module/frogr">Damnes Lies platform</a>, without having to worry about the strings changing again before the release.

Another thing I'd like to publicly say here and now, and which you might have already noticed from the previous paragraph, is that this release is gonna be the first one after having moved <a title="Frogr web site" href="http://live.gnome.org/Frogr">frogr</a> to the <a title="The GNOME project" href="http://www.gnome.org">GNOME</a> infrastructure, which basically means that its <a title="Previous (deprecated) site for frogr at Google Code" href="http://code.google.com/p/frogr/">previous site and mailing lists at Google Code</a>, as well as the <a title="Previous (deprecated) git repository for Frogr at gitorious" href="http://gitorious.org/frogr">old repository at gitorious</a>, are now deprecated in favour of <a title="live.gnome.org WIKI" href="http://live.gnome.org">live.gnome.org</a>, <a title="GNOME bugzilla: frogr" href="https://bugzilla.gnome.org/browse.cgi?product=frogr">GNOME's bugzilla</a> and <a title="Frogr's git repository at GNOME" href="http://git.gnome.org/browse/frogr/">GNOME's git repository</a>. And this, at least in my very personal opinion, is really great news for <a title="Frogr web site" href="http://live.gnome.org/Frogr">frogr</a>, and I'm really happy about it.

Other than that, but related, last week I've finally decided to apply for the <a title="The GNOME Foundation" href="http://foundation.gnome.org/">GNOME Foundation</a>, and got accepted, which was also great news for me, since it was something already in the back of my mind for some time, and I guess I just needed a 'trigger' for daring to apply for it... and moving <a title="Frogr web site" href="http://live.gnome.org/Frogr">frogr</a> to <a title="The GNOME project" href="http://www.gnome.org">GNOME</a>, together with all the a11y related work I've been doing during the past months in <a title="WebKitGTK+ website" href="http://webkitgtk.org">WebKitGTK+</a>, as part of the <a title="WebKit" href="http://www.webkit.org">WebKit</a> team at <a title="Igalia" href="http://www.igalia.com">Igalia</a>, were actually good 'triggers' for that.

So a big 'thank you' to all who helped making all that possible, specially to <a title="Claudio Saavedra's blog" href="http://people.gnome.org/~csaavedra">Claudio Saavedra</a> for encouraging for doing this moves, and to Christer Edwards, who kindly attended all my continuous requests to perform all the related tasks in record time.

By the way, as <a title="Claudio Saavedra's blog" href="http://people.gnome.org/~csaavedra">Claudio</a> and some other workmates, I'll be attending to <a title="Free and Open Source Developers European Meeting" href="http://www.fosdem.org">FOSDEM</a> once again this year thanks to the support from <a title="Igalia" href="http://www.igalia.com">Igalia</a>, where I'm starting to have a hard time sometimes to explain that the Beer Event has nothing to do with me willing to go every year :-).

Anyway, apart from hanging around the venue, I'll also be giving a talk about a11y in <a title="WebKitGTK+ website" href="http://webkitgtk.org">WebKitGTK+</a> in the <a title="FOSDEM 2011's devrooms list" href="http://fosdem.org/2011/devrooms-list">accessibility devroom</a> on Sunday, so that makes another pretty good reason for me to go this year. So, you see? It's not only about beer!

<strong>Update [05/02/2011]:</strong> At the end, and due to unexpected and very important personal matters, I was finally not able to attend FOSDEM. Hopefully I'll be able to go next year, but as for now I'd like to wish all the attendants of the 2011 edition have a great time there and enjoy this awesome meeting, which is one of my favorite ones every year.
