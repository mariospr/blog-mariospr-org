<div class="software-section">
<h1>Chromium &amp; Blink</h1>
<a href="http://www.chromium.org/Home">Chromium</a> is an Open Source browser and <a href="http://www.chromium.org/blink">Blink</a> is the web rendering engine powering it behind the scenes, being both widely used across different types of platforms and devices such as Android, iOS, Linux, Mac OS X and Windows.

As part of <a title="Samsung's Open Source Group" href="http://blogs.s-osg.org/">Samsung's Open Source Group</a>, back in 2014, I briefly contributed to the Blink web engine, with a focus on text layout and rendering, while mentoring other members of the team.

Nowadays, as a (returning) member of <a href="https://www.igalia.com">Igalia</a> and part of its <a href="https://www.igalia.com/browsers-webkit-chromium">Chromium team</a>, I contribute to Chromium in a more permanent basis, starting with the <a href="https://www.chromium.org/servicification">Chromium Servicification Project</a> by the end of 2018.

<a href="http://www.chromium.org/">Check out its website for more information</a>
<h1>Endless OS</h1>
<a href="https://endlessm.com/developer/#row-1">Endless OS</a> is a Linux-based Operating System aimed at providing access to technology to the next billion people in the developing world. It relies on several <a href="https://github.com/endlessm">Open Source projects we contribute back to</a>, as well as on <a href="https://endlessm.com/developer/#row-3">other projects specifically created for this platform</a>.

As a member of <a href="http://endlessm.com">Endless</a>'s <em>core team</em>, I worked on any part of the stack I can contribute to: from lower level elements such as <a href="https://github.com/endlessm/systemd/commits/master?author=mariospr">systemd</a>, <a href="https://github.com/endlessm/system-config-printer/commits/master?author=mariospr">printing</a> <a href="https://github.com/endlessm/eos-config-printer/commits/master?author=mariospr">configuration</a>, <a href="http://flatpak.org">flatpak</a> or the <a href="https://github.com/endlessm/pulseaudio/commits/master?author=mariospr">audio subsystem</a>, to higher level components such as <a href="http://www.webkit.org">WebKit</a>, <a href="http://www.chromium.org/Home">Chromium</a> or the <a href="https://github.com/endlessm/eos-desktop/commits/master?author=mariospr">Desktop Shell</a>.

<a href="https://endlessm.com/developer/#row-1">Check out its website for more information</a>

<h1>Samsung's Smart TV platform</h1>
<a href="http://www.samsungdforum.com/Features/Introduction">Samsung's Smart TV</a> is a platform that integrates Internet and web features in TVs and set-top boxes, as well as provides a way for technological convergence between those TVs and other devices like computers, smartphones and tablets.

I worked on performance optimizations and infrastructure related improvements in the context of <a href="http://www.webkit.org">WebKit</a> and the <a href="http://www.samsungdforum.com/Features/Introduction">Smart TV platform</a> as part of <a href="http://www.samsung.com/uk/home/">Samsung Research UK</a>'s Smart TV team.

<h1>WebKit Watcher</h1>
WebKit Watcher is an <a title="Android" href="http://www.android.com/">Android</a> application to monitor the <a title="The WebKit project" href="http://www.webkit.org">WebKit</a> <a title="WebKit Buildbots" href="http://build.webkit.org">buildbots</a>. I developed this small app with <a title="Android" href="http://www.android.com/">Android</a> 2.2 (Froyo) and a HTC Nexus One in mind, but should work fine in any other Android based phone with at least the 1.5 version of the Android platform.

Grab the APK file <a title="WebKit Watcher's APK file" href="/misc/WebKitWatcher.apk">from here</a>, or right <a title="WebKit Watcher on Google's Play Store" href="https://play.google.com/store/search?q=webkit+watcher">from Google's Play Store</a>.

<a href="https://github.com/mariospr/webkit-watcher">Check out the source code from its git repository</a>

<h1>WebKit &amp; WebKitGTK+</h1>
<a href="http://www.webkit.org">WebKit</a> is an Open Source web browser engine, and also the name of the Mac OS X system framework version of the portable engine that's used by Safari, Dashboard, Mail, and many other OS X applications.

In the past, I mainly hacked on <a href="http://www.webkitgtk.org">WebKitGTK+</a>, the port of WebKit to the GTK+ platform as part of <a href="https://www.igalia.com">Igalia</a>'s <a href="https://www.igalia.com/browsers-webkit-chromium">WebKit team</a>, mostly improving AT-SPI/ATK based accessibility support and implementing the new API based in <a href="trac.webkit.org/wiki/WebKit2">WebKit2</a>. After that, I continued to contribute to the project when I joined <a href="http://www.samsung.com/uk/home/">Samsung Research UK</a>, where I achieved the status of <a href="https://www.webkit.org/coding/commit-review-policy.html">WebKit Reviewer</a> because of my work on the accessibility implementation in <a href="http://www.webkitgtk.org">WebKitGTK+</a>.

<a href="http://www.webkit.org">Check out its website for more information</a>

<h1>epiphany: the GNOME browser</h1>
Epiphany is the web browser for the GNOME desktop. Its goal is to be simple and easy to use. Epiphany ties together many GNOME components in order to let you focus on the Web content, instead of the browser application.

As part of the GNOME project, Epiphany is Free Software. In epiphany, and as a member of<a href="https://www.igalia.com"> Igalia</a>'s <a href="https://www.igalia.com/browsers-webkit-chromium">WebKit team</a>, I worked mainly on a few bugfixing tasks as well as helping port some parts of the code to the new WebKit2GTK+ API, which I also helped to write.

<a href="http://projects.gnome.org/epiphany/">Check out its website for more information</a>

<h1>frogr: a flickr remote organizer for GNOME</h1>
<a title="Frogr website at GNOME.org" href="https://wiki.gnome.org/Apps/Frogr">Frogr</a> is a small <a href="http://www.gtk.org">GTK+</a> application for the GNOME desktop that allows users to manage their accounts in the Flickr image hosting website. It supports all the basic Flickr features, including uploading pictures, adding descriptions, setting tags and managing sets and groups pools.

I am the lead developer of this project, which I started working on back in 2009, and so far I'm fairly happy with its current status and the implemented features it provides, specially considering that I only work on it every now and then in my spare time, a really scarce asset nowadays. This project is Free Software and published under the terms of the <a href="http://www.gnu.org/licenses/gpl-3.0.html">GNU General Public License v3</a>.

<a href="https://wiki.gnome.org/Apps/Frogr">Check out its website for more information</a>

<h1>Vagalume: Client for Last.fm and compatible services</h1>
Vagalume is a Last.fm client written using GTK+. It should work on any GNOME-based system, but it was originally designed work on Maemo, the platform used by some Nokia devices such as the 770, N800, N810 and N900.

My main contributions to this project are the implementation of the Maemo status bar plugin, a freedesktop tray icon for the desktop, a vagalumectl script for remote control capabilities through D-Bus, and "media keys support" for the freedesktop version, using the gnome-settings-daemon.

<a href="http://vagalume.igalia.com/">Check out its website for more information</a>

<h1>Litl OS</h1>
<a href="http://litl.com/webbook/essays/software.htm">Litl OS</a> is a cloud-based operating system designed an implemented specifically for the <a href="http://litl.com/webbook/meet-webbook/overview.htm">Litl Webbook</a>, which as been <a href="http://www.coolthings.com/litl-webbook-beats-chromeos-becomes-first-cloud-computer/">described as the "world's first" true cloud-based netbook</a>, released even before than the well-known ChromeOS, from Google.

<h1>Hildon Application Manager &amp; Maemo's APT</h1>
The Hildon Application Manager is a program to manage add-on software components for the Hildon Desktop. It uses the Debian package management tools provided by the Maemo platform (namely APT and dpkg) and presents a ‘end user oriented’ interface to them.

On this project, I worked both on bugfixing, development of some new features and also helping co-maintain these two projects for some time as well along with my colleagues at <a href="https://www.igalia.com">Igalia</a>, being responsible of publishing regular new releases.

<h1>PhpReport</h1>
PhpReport is a web application for time tracking, which provides useful statistics about users, projects, and more. The application is focused on companies or any situation where multiple people collaborate on a project. PhpReport allows managers to see project-level statistics such as cost and deviation, as well as user-level statistics like number of days worked and unused vacation days.

I worked in the first version of this project as one of my first assignments at <a href="https://www.igalia.com">Igalia</a>, mainly by implementing new features such as customers management, improved projects and users management, support for charts support both for users and projects and improved tasks management and "new tasks" form.

<a href="http://phpreport.igalia.com">Check out its website for more information</a>

<h1>eFortunes</h1>
If you mix functional programming (using <a href="http://www.erlang.org/">Erlang</a>) with web apps development (using <a href="http://yaws.hyber.org/">Yaws</a>) and a fault-tolerant storage system (using Mnesia) for your preferred fortunes... what do you get? Yeah!... eFortunes: a distributed, fault-tolerant and scalable fortunes server developed in Erlang.

Nowadays eFortunes is just an abandoned project that used to be useful for me to learn more about web development with erlang, and the fortunes server was just an excuse for me to programming such that workbench.

<a title="efortunes git repository at github" href="https://github.com/mariospr/efortunes">Check out the source code from its git repository</a>
